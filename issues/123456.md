### Title
Missing Dockerfiles for production deployment of Backend and Storefront

### Description
The repository lacks Dockerfile definitions for the backend and storefront applications. The existing docker-compose.yml is only configured to spin up local infrastructure (PostgreSQL and Redis) for development.

### Why it matters:
Without containerized application services, the deployment process is fragile, highly dependent on the host environment, and not ready for modern scalable orchestration (e.g., AWS ECS, Kubernetes). This is a blocker for reliable production deployments.

### Suggested direction:
Create production-optimized Dockerfiles for the Storefront (leveraging Next.js standalone output) and the Backend. Also provide a docker-compose.prod.yml or equivalent deployment manifests.
